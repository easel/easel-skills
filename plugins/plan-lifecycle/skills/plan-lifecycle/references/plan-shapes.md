# Plan Shapes

Use the smallest shape that gives the user enough control to proceed.

## Implementation Plan

```markdown
## Goal
## Scope
## Assumptions
## Work Breakdown
## Validation
## Risks
## Open Questions
## Handoff
```

## Release Plan

```markdown
## Release Target
## Current State
## Version And Metadata
## Validation
## Release Steps
## Authorization Boundaries
## Rollback Or Recovery
## Open Questions
```

## Test Plan

```markdown
## Goal
## Risk Model
## Coverage
## Test Cases
## Fixtures And Environment
## Commands And Evidence
## Gaps
## Acceptance Criteria
```

## Research-To-Plan

```markdown
## Decision
## Evidence Summary
## Chosen Approach
## Work Breakdown
## Validation
## Risks
## Sources
## Open Questions
```

## Data Plan

```markdown
## Data Contract
## Source Evidence
## Grain And Keys
## Mapping Or Documentation Scope
## Quality Gates
## Review And Ownership
## Open Questions
```

## Post-Implementation Review

```markdown
## Original Plan
## Delivered Work
## Deviations
## Validation Evidence
## Remaining Risks
## Follow-Ups
## Release Or Documentation Notes
```

## Task Format

Use this compact shape for breakdowns intended for agents:

```markdown
- Task:
  - Objective:
  - Inputs:
  - Files or systems:
  - Steps:
  - Acceptance:
  - Validation:
  - Non-scope:
  - Dependencies:
```
